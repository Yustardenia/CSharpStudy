# 9.回文数

## 题目类型

- 数学
- 回文判断
- 边界处理

## 我这次代码的问题

你原来写的是：

```csharp
public class Solution {
    public bool IsPalindrome(int x)
    {
        string s = x.ToString();
        var s1 = s.Reverse();
        return s == s1;
    }
}
```

主要问题在这里：

```csharp
var s1 = s.Reverse();
```

- `s.Reverse()` 返回的不是 `string`
- 它返回的是 `IEnumerable<char>`
- 所以后面这句：

```csharp
return s == s1;
```

- 比较的不是两个字符串内容
- 而是 `string` 和 `IEnumerable<char>`，这样当然不对

## 如果只修正字符串写法

```csharp
using System.Linq;

public class Solution
{
    public bool IsPalindrome(int x)
    {
        string s = x.ToString();
        string reversed = new string(s.Reverse().ToArray());
        return s == reversed;
    }
}
```

## 更推荐的进阶解法

题目进阶要求是不把整数转成字符串，所以更推荐数学解法。

核心思路：

- 负数一定不是回文
- 如果一个正数最后一位是 `0`，但它本身不是 `0`，那它也不可能是回文
- 不需要把整个数字都反转，只需要反转后一半
- 最后比较前一半和后一半是否相等

## 最优解

```csharp
public class Solution
{
    public bool IsPalindrome(int x)
    {
        if (x < 0 || (x % 10 == 0 && x != 0))
        {
            return false;
        }

        int reversedHalf = 0;

        while (x > reversedHalf)
        {
            int digit = x % 10;
            reversedHalf = reversedHalf * 10 + digit;
            x /= 10;
        }

        return x == reversedHalf || x == reversedHalf / 10;
    }
}
```

## 为什么这样写

### 1. 负数直接返回 `false`

```csharp
if (x < 0)
{
    return false;
}
```

- 例如 `-121`
- 从左往右读是 `-121`
- 从右往左读会变成 `121-`
- 不可能一样

### 2. 末尾是 `0` 的数也要特判

```csharp
(x % 10 == 0 && x != 0)
```

- 例如 `10`
- 如果它是回文，反过来应该还是 `10`
- 但倒过来其实像 `01`
- 所以不可能是回文
- 唯一例外是 `0` 本身，它是回文

### 3. 为什么只反转一半

正常反转整个整数也能做，但可能有溢出风险。

这题更巧妙的地方是：

- 如果一个数是回文
- 那么前一半和后一半倒过来应该一样

例如 `1221`：

- 前半：`12`
- 后半反转后：`12`

例如 `12321`：

- 前半：`12`
- 中间：`3`
- 后半反转后：`123`

中间那个 `3` 不影响回文判断，所以最后把它去掉再比较。

### 4. 这段循环在做什么

```csharp
while (x > reversedHalf)
{
    int digit = x % 10;
    reversedHalf = reversedHalf * 10 + digit;
    x /= 10;
}
```

每轮做三件事：

- `digit = x % 10`
  - 取出原数字最后一位
- `reversedHalf = reversedHalf * 10 + digit`
  - 把这一位拼到“后半部分的反转结果”里
- `x /= 10`
  - 把原数字最后一位去掉

循环条件是：

```csharp
x > reversedHalf
```

意思是：

- 只要前半部分还比后半部分长，就继续拆
- 一旦后半部分长度追上前半部分，就可以停了

## 最后为什么这样比较

```csharp
return x == reversedHalf || x == reversedHalf / 10;
```

这是为了同时处理偶数位和奇数位。

### 偶数位

例如 `1221`

循环结束后：

- `x = 12`
- `reversedHalf = 12`

所以直接比较：

```csharp
x == reversedHalf
```

### 奇数位

例如 `12321`

循环结束后：

- `x = 12`
- `reversedHalf = 123`

这时中间那个 `3` 是多出来的，不影响回文判断，所以去掉：

```csharp
reversedHalf / 10 = 12
```

所以比较：

```csharp
x == reversedHalf / 10
```

## 复杂度

- 时间复杂度：`O(log10 n)`
- 空间复杂度：`O(1)`

## 复盘

- 看到“整数回文”先想数学解法，不要先想字符串。
- `Reverse()` 要记住返回的是字符序列，不是字符串。
- 这题最值得记住的是“只反转一半”这个技巧。

## 一句话记忆

- 负数和末尾 0 先排掉，然后只反转后一半，最后和前一半比较。
