# 10.正则表达式匹配

## 题目类型

- 动态规划
- 字符串匹配
- 分类讨论

## 我当前的状态

- 这题目前不会做。
- 现在已经知道这题经典解法是动态规划。
- 当前最需要建立的是对 `dp[i][j]` 的理解，以及 `*` 为什么会分成两种转移。

## 一个很容易冒出来的问题

### 题目名字叫“正则表达式匹配”，为什么不能直接用正则库？

技术上当然可以写出类似这样的代码：

```csharp
using System.Text.RegularExpressions;

public class Solution
{
    public bool IsMatch(string s, string p)
    {
        return Regex.IsMatch(s, "^" + p + "$");
    }
}
```

但是这题不应该这样做。

## 为什么不应该直接调用正则库

- 这题真正想考的不是“会不会调库”。
- 它想考的是：
  - 你能不能理解 `.` 和 `*` 的匹配规则
  - 你能不能自己实现这个匹配过程
  - 你会不会设计状态和转移

换句话说，这题虽然名字里有“正则表达式”，但本质上是在让你自己实现一个简化版的匹配器。

## 为什么调库不算这题的正解

- 这就像：
  - 让你实现排序
  - 结果你直接调用 `Array.Sort()`
- 或者：
  - 让你实现二叉树遍历
  - 结果你直接调一个现成工具函数

题目真正要考的核心逻辑，被库直接替你做掉了。

## 这题真正考什么

- 普通字符怎么匹配
- `.` 怎么匹配任意单个字符
- `x*` 为什么不是“只匹配一个字符”，而是“前一个字符可重复 0 次到很多次”
- 如何把这个过程变成动态规划或记忆化搜索

## 这题最难的点

不是语法，而是：

- `*` 不是单独匹配自己
- `*` 是在控制它前面那个字符重复多少次

也就是说：

- `a*` 可以表示：
  - 空串
  - `a`
  - `aa`
  - `aaa`
  - ...

所以在动态规划里，它天然就会分成两种情况：

- 把 `a*` 当成出现 `0` 次
- 把 `a*` 当成还能继续吃一个字符

## 动态规划状态定义

这题最关键的一句是：

```text
dp[i][j] 表示 s 的前 i 个字符，能不能被 p 的前 j 个字符完全匹配
```

最后答案就是：

```text
dp[s.Length][p.Length]
```

## 普通字符和 '.' 怎么转移

如果当前 `p[j - 1]` 是普通字符，或者是 `.`：

- 只有当前字符能匹配上时，才有机会成功
- 这时直接看左上角：

```text
dp[i][j] = dp[i - 1][j - 1]
```

## '*' 怎么转移

如果当前 `p[j - 1] == '*'`，那它控制的是 `p[j - 2]`。

这时要分两种情况：

### 1. 前一个字符出现 0 次

直接跳过这两个字符：

```text
dp[i][j] = dp[i][j - 2]
```

### 2. 前一个字符出现至少 1 次

前提是：

- `s[i - 1]` 能和 `p[j - 2]` 匹配

这时：

```text
dp[i][j] = dp[i - 1][j]
```

因为：

- 这次 `*` 吃掉了一个字符
- 但 `*` 还可以继续吃后面的字符
- 所以模式位置不动，字符串缩短一位

## '*' 的核心记忆

看到 `x*` 就立刻想到两条路：

- 不用它：`dp[i][j - 2]`
- 用它继续吃：`dp[i - 1][j]`

## 标准 C# 写法

```csharp
public class Solution
{
    public bool IsMatch(string s, string p)
    {
        int m = s.Length;
        int n = p.Length;
        bool[,] dp = new bool[m + 1, n + 1];

        dp[0, 0] = true;

        for (int j = 2; j <= n; j++)
        {
            if (p[j - 1] == '*')
            {
                dp[0, j] = dp[0, j - 2];
            }
        }

        for (int i = 1; i <= m; i++)
        {
            for (int j = 1; j <= n; j++)
            {
                if (p[j - 1] == '.' || p[j - 1] == s[i - 1])
                {
                    dp[i, j] = dp[i - 1, j - 1];
                }
                else if (p[j - 1] == '*')
                {
                    dp[i, j] = dp[i, j - 2];

                    if (p[j - 2] == '.' || p[j - 2] == s[i - 1])
                    {
                        dp[i, j] = dp[i, j] || dp[i - 1, j];
                    }
                }
            }
        }

        return dp[m, n];
    }
}
```

## 复盘

- 这题虽然名字叫“正则表达式匹配”，但不是让你调正则库。
- 真正要练的是：
  - 状态定义
  - 分类讨论
  - `*` 的两条转移
- 以后看到这题，先记住一句：
  - `*` 控制的是前一个字符，不是它自己。

## 一句话记忆

- 这题不是“会不会正则”，而是“会不会自己实现一个简化版正则匹配器”。
