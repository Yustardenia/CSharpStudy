# 17.电话号码的字母组合

## 题目类型

- 字符串
- 迭代构造
- 分层扩展

## 我的当前状态

- 这题一开始会先想到用字典保存数字到字母的映射。
- 这个方向没有问题。
- 当前更适合先掌握“不写递归”的迭代 / 循环版，把“每一层怎么扩展结果”想清楚。

## 先想清楚这题到底在做什么

假设输入是：

```text
"23"
```

那么：

- `2 -> "abc"`
- `3 -> "def"`

题目不是让我们分别拿到：

```text
"abc"
"def"
```

就结束，而是要把它们组合起来，得到：

```text
["ad","ae","af","bd","be","bf","cd","ce","cf"]
```

所以本质上是在做：

- 第一位数字给你一组选择
- 第二位数字再给你一组选择
- 把前一层已有结果，和当前这一层字母全部拼起来

## 为什么字典可以用

这题用字典保存映射很自然：

```csharp
Dictionary<char, string> dict = new Dictionary<char, string>()
{
    {'2', "abc"},
    {'3', "def"},
    {'4', "ghi"},
    {'5', "jkl"},
    {'6', "mno"},
    {'7', "pqrs"},
    {'8', "tuv"},
    {'9', "wxyz"}
};
```

这样每次只要拿到当前数字，就能立刻知道它对应哪一组字母。

这题里：

- 用字典是可以的
- 也可以用数组映射

但当前阶段更重要的不是“字典还是数组”，而是“怎么生成所有组合”。

## 我现在先记住的版本：迭代 / 循环版

```csharp
public class Solution
{
    public IList<string> LetterCombinations(string digits)
    {
        List<string> ans = new List<string>();
        if (string.IsNullOrEmpty(digits))
        {
            return ans;
        }

        Dictionary<char, string> dict = new Dictionary<char, string>()
        {
            {'2', "abc"},
            {'3', "def"},
            {'4', "ghi"},
            {'5', "jkl"},
            {'6', "mno"},
            {'7', "pqrs"},
            {'8', "tuv"},
            {'9', "wxyz"}
        };

        ans.Add("");

        foreach (char digit in digits)
        {
            string letters = dict[digit];
            List<string> temp = new List<string>();

            foreach (string prefix in ans)
            {
                foreach (char c in letters)
                {
                    temp.Add(prefix + c);
                }
            }

            ans = temp;
        }

        return ans;
    }
}
```

## 这版代码每一段在做什么

### 1. 先准备结果列表

```csharp
List<string> ans = new List<string>();
```

- `ans` 表示“当前已经生成好的所有组合”。
- 注意它一开始不是最终答案，而是会一层一层变化。

### 2. 空字符串特判

```csharp
if (string.IsNullOrEmpty(digits))
{
    return ans;
}
```

- 如果没有数字，就没有任何组合。
- 所以直接返回空列表。

### 3. 用字典保存映射

```csharp
Dictionary<char, string> dict = ...
```

- 当前数字是什么，就能查到它对应的一组字母。

例如：

- `'2' -> "abc"`
- `'7' -> "pqrs"`

### 4. 先放一个空串进去

```csharp
ans.Add("");
```

这一句特别关键。

为什么一开始不是空列表，而是放一个 `""`？

因为后面每一轮都要做拼接。

比如第一轮处理 `'2'` 时：

- `"" + 'a' = "a"`
- `"" + 'b' = "b"`
- `"" + 'c' = "c"`

如果一开始 `ans` 是空列表，那后面根本没有前缀可以拼。

### 5. 外层循环：按数字一位一位处理

```csharp
foreach (char digit in digits)
```

- 每次处理一个数字。
- 可以理解为“逐层扩展答案”。

### 6. 取当前数字对应的字母组

```csharp
string letters = dict[digit];
```

例如：

- 当前 `digit = '2'`
- 那么 `letters = "abc"`

### 7. 新建一个临时列表

```csharp
List<string> temp = new List<string>();
```

- `temp` 用来存“处理完当前数字之后的新结果”。
- 不能直接一边遍历 `ans` 一边改 `ans`，所以用 `temp` 接住这一轮的新组合。

### 8. 遍历当前已有的每个前缀

```csharp
foreach (string prefix in ans)
```

- `prefix` 表示前面已经拼好的部分。
- 比如处理到第二位时，`prefix` 可能是 `"a"`、`"b"`、`"c"`。

### 9. 再遍历当前数字对应的每个字母

```csharp
foreach (char c in letters)
```

- 当前这一位有哪些字母可选，就一个个试。

### 10. 拼成新组合

```csharp
temp.Add(prefix + c);
```

- 当前已有前缀 `prefix`
- 加上当前字母 `c`
- 就得到一个新的组合

### 11. 一轮结束后更新结果

```csharp
ans = temp;
```

- 表示“当前这一位数字已经处理完了”
- 以后下一轮就基于这些新结果继续扩展

### 12. 全部处理完后返回

```csharp
return ans;
```

- 当所有数字都处理完时，`ans` 里就是最终答案。

## 用 `"23"` 手推一遍

### 初始

```text
ans = [""]
```

### 处理 `'2'`

```text
letters = "abc"
```

拿当前 `ans` 里的空串去拼：

- `"" + a = "a"`
- `"" + b = "b"`
- `"" + c = "c"`

所以这一轮结束后：

```text
ans = ["a","b","c"]
```

### 再处理 `'3'`

```text
letters = "def"
```

现在旧的 `ans` 是：

```text
["a","b","c"]
```

逐个扩展：

- `"a"` 拼出：`"ad","ae","af"`
- `"b"` 拼出：`"bd","be","bf"`
- `"c"` 拼出：`"cd","ce","cf"`

所以最终：

```text
["ad","ae","af","bd","be","bf","cd","ce","cf"]
```

## 为什么这版适合现在记

- 不需要先掌握回溯和撤销选择。
- 只要抓住一句话：

```text
已有结果 × 当前字母组 = 新一轮结果
```

这题就能写出来。

## 复杂度

假设输入长度是 `n`。

- 每一位最多有 `4` 个字母
- 最终组合数量大约是 `O(4^n)`

所以：

- 时间复杂度：`O(4^n * n)` 量级
- 空间复杂度：`O(4^n * n)` 量级

这是因为题目本身就要求把所有组合都生成出来。

## 复盘

- 这题最重要的不是字典，而是“分层扩展”。
- 以后看到“每一位都有若干种选择，要求所有组合”，就可以先想：
  - 当前已有结果是什么
  - 新的一层选择是什么
  - 怎么把它们两两拼接

## 一句话记忆

- 每读到一个数字，就把当前所有前缀和这一组字母全部拼一遍，一层一层扩成最终答案。
