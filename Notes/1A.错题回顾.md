# 错题回顾

## 需要回顾的题目

### 12.整数转罗马数字

- 当前状态：这题第一时间不会做，容易被一堆规则吓住，暂时还没把“先列固定表，再从大到小处理”变成自己的第一反应。
- 主要问题：
  - 容易把注意力放在 `4、9、40、90、400、900` 这些特殊情况上，结果越想越乱。
  - 还不够习惯把固定规则题先整理成映射表。
  - 写代码时边界不够敏感，把 `>=` 写成了 `>`，导致刚好相等时无法正确处理。
- 回顾重点：
  - 先记住所有合法组合本来就只有固定这些：`M, CM, D, CD, C, XC, L, XL, X, IX, V, IV, I`
  - 本质上是在“照表减数”，不是在现场推规则。
  - 遇到这种题，优先检查是不是可以“列规则表 + 顺序处理”。

### 10.正则表达式匹配

- 当前状态：这题目前不会，已经知道不能靠正则库取巧，核心要学的是动态规划状态定义和 `*` 的转移。
- 主要问题：
  - 容易被题目名字误导，先想到直接调正则库。
  - 还没有完全建立 `dp[i][j]` 的直觉。
  - 对 `*` 为什么要分成“0 次”和“继续吃字符”两种情况还不够熟。
- 回顾重点：
  - `dp[i][j]` 表示前缀匹配
  - 普通字符 / `.` 看左上角 `dp[i - 1][j - 1]`
  - `x*` 看 `dp[i][j - 2]` 和 `dp[i - 1][j]`

### 9.回文数

- 当前状态：已经知道字符串法哪里写错了，但更值得掌握的是“不转字符串、只反转一半”的数学解法。
- 主要问题：
  - 容易忘记 `Reverse()` 返回的是 `IEnumerable<char>`，不是字符串。
  - 容易直接拿 `string` 和反转后的字符序列比较。
  - 还不够熟悉“只反转一半”这种更巧妙的整数题技巧。
- 回顾重点：
  - 负数直接不是回文
  - 末尾是 `0` 且本身不是 `0` 的数也不是回文
  - 循环里只反转后一半
  - 结尾比较 `x == reversedHalf` 或 `x == reversedHalf / 10`

### 8.字符串转换整数 (atoi)

- 当前状态：这题已经能独立写出通过解法，当前重点不是“会不会”，而是把写法模板化、提高可读性。
- 主要问题：
  - 第一版更偏 `foreach` 加状态控制，结构上不如下标扫描清晰。
  - 容易用 `Parse` 做字符转数字，其实 `c - '0'` 更直接。
  - 需要继续熟悉正负号和溢出判断如何统一成一套扫描模板。
- 回顾重点：
  - 按阶段处理：空格、符号、数字、截断
  - 用下标 `i` 扫描字符串通常更自然
  - 记住边界：正数到 `7`，负数到 `8`

### 7.整数反转

- 当前状态：已经能想到字符串反转加 `TryParse`，但还没有完全吃透数学最优解，尤其是溢出判断的两个 `if`。
- 主要问题：
  - 第一反应容易先写字符串解法，没有先想到 `% 10` 和 `/ 10`。
  - 容易把“输入范围”和“反转结果范围”混在一起。
  - 对为什么要在 `result = result * 10 + digit` 之前判断溢出，还不够熟。
  - 对边界数字 `7` 和 `-8` 的来源不够敏感。
- 回顾重点：
  - 每轮先取最后一位：`digit = x % 10`
  - 每轮去掉最后一位：`x /= 10`
  - 每轮拼回结果：`result = result * 10 + digit`
  - 拼之前先做正负两侧的边界判断

### 6.Z字形变换

- 当前状态：这题目前不会，容易被“Z 字形图案”带偏，还没建立起“按行模拟”的直觉。
- 主要问题：
  - 容易先去想怎么还原整个图形，而不是直接按行收集字符。
  - 对“方向切换”类模拟题不熟，没能立刻想到维护 `currentRow` 和 `goingDown`。
  - 特殊情况意识不够强，容易漏掉 `numRows == 1` 和 `numRows >= s.Length`。
  - 自己写代码时，把 `nowRow` 和 `goDown` 放进了循环内部，导致每轮都被重置。
  - 把 `rows[nowRow]` 写成了 `rows[c]`，混淆了“字符”和“行号”。
- 回顾重点：
  - 最终目标不是画图，而是得到“每一行的字符”。
  - 用 `numRows` 个字符串分别存每一行。
  - 每次走到顶行或底行时切换方向。
  - 凡是依赖上一轮状态的变量，先检查是不是应该定义在循环外。

### 5.最长回文子串

- 当前状态：正在重建手感，已经明确更适合用“中心扩展”而不是“从字符串两端整体夹逼”。
- 主要问题：
  - 容易把“当前中心”和“左右扩展指针”混在一起，导致循环变量失控。
  - `Substring(start, length)` 的第二个参数容易误写成右边界。
  - 退出扩展时，合法回文区间和失败比较区间容易混淆。
  - 容易漏掉偶数回文中心。
- 回顾重点：
  - 奇数中心 `Expand(i, i)`。
  - 偶数中心 `Expand(i, i + 1)`。
  - 扩展结束后，如何正确记录回文左右边界和长度。
