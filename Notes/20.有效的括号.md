# 20.有效的括号

## 题目类型

- 字符串
- 栈

## 我的第一反应

- 看到括号匹配，能感觉到“要一对一对消掉”，但不太会用栈。
- 对“栈到底在这题里装什么”没有直觉。

## 我的难点

- 不熟悉栈的使用场景，不知道为什么这题适合用栈。
- 不容易立刻想到：右括号应该匹配“最近一个还没配对的左括号”。
- 容易把问题想成“找有没有对应括号”，但忽略了顺序。

## 我这次的错误

- 还没真正写代码，当前主要是思路卡住。
- 对栈的核心作用理解不够明确：栈里装的不是所有字符，而是“还没匹配掉的左括号”。

## 正确思路

- 遍历字符串。
- 遇到左括号，就把它压入栈中。
- 遇到右括号，就检查栈顶是不是对应类型的左括号。
- 如果栈为空，或者类型不匹配，直接返回 `false`。
- 遍历结束后，如果栈为空，说明全部匹配成功；否则还有没闭合的左括号，返回 `false`。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 复盘

- 这题里栈的作用可以记成一句话：`栈里装的是还没匹配掉的左括号。`
- “最近打开的括号要先关掉” 这一点天然对应栈的后进先出。
- 栈题的常见检查顺序：
  - 当前是右括号时，先判断栈是否为空。
  - 再判断栈顶是否匹配。
  - 最后遍历结束后，再判断栈是否为空。

## 最优解

- 对这题来说，最优解就是标准栈解法。它既最好理解，也是这类括号匹配题最稳定的模板。

```csharp
public class Solution
{
    public bool IsValid(string s)
    {
        Stack<char> stack = new Stack<char>();

        foreach (char c in s)
        {
            if (c == '(' || c == '[' || c == '{')
            {
                stack.Push(c);
            }
            else
            {
                if (stack.Count == 0)
                {
                    return false;
                }

                char top = stack.Pop();

                if (c == ')' && top != '(') return false;
                if (c == ']' && top != '[') return false;
                if (c == '}' && top != '{') return false;
            }
        }

        return stack.Count == 0;
    }
}
```
