# 16.最接近的三数之和

## 题目类型

- 排序
- 双指针
- 贪心调整
- 剪枝

## 我的当前状态

- 这题已经知道标准解法是：
  - 排序
  - 固定一个数
  - 后面两个数用双指针
- 当前最需要记住的是：
  - 更新答案时看“差值”
  - 移动指针时看“sum 和 target 的大小关系”
- 另外，这题还能在标准双指针基础上继续做剪枝优化。

## 我一开始容易犯的错

### 1. 把 `sum` 和 “差值”混在一起

- 题目要返回的是：
  - 最接近 `target` 的那个三数之和
- 不是返回最小差值

所以要么：

- 直接保存当前最优和 `best`

要么：

- 同时保存 `minDiff` 和 `ans`

但不能把这两个概念混着用。

### 2. 用“差值大小”决定指针移动方向

这不对。

- 差值只能告诉你“离得远不远”
- 不能告诉你“该往左走还是往右走”

真正决定指针移动方向的，是：

- `sum < target` 还是 `sum > target`

因为：

- `sum < target`，说明当前和太小，要让和变大，所以 `left++`
- `sum > target`，说明当前和太大，要让和变小，所以 `right--`

## 标准最优解

```csharp
public class Solution
{
    public int ThreeSumClosest(int[] nums, int target)
    {
        Array.Sort(nums);
        int best = nums[0] + nums[1] + nums[2];

        for (int i = 0; i < nums.Length - 2; i++)
        {
            int left = i + 1;
            int right = nums.Length - 1;

            while (left < right)
            {
                int sum = nums[i] + nums[left] + nums[right];

                if (Math.Abs(sum - target) < Math.Abs(best - target))
                {
                    best = sum;
                }

                if (sum < target)
                {
                    left++;
                }
                else if (sum > target)
                {
                    right--;
                }
                else
                {
                    return sum;
                }
            }
        }

        return best;
    }
}
```

## 为什么这是最优思路

- 排序后，数组有序
- 固定一个数后，剩下两个数可以用双指针在 `O(n)` 内完成搜索
- 外层枚举 `i`，内层双指针，总复杂度就是：

```text
O(n^2)
```

这已经是这题公认的最优级别解法。

## 标准版里最重要的两条规则

### 1. 更新答案时看差值

```csharp
if (Math.Abs(sum - target) < Math.Abs(best - target))
{
    best = sum;
}
```

- 谁离 `target` 更近，就保留谁

### 2. 移动指针时看方向

```csharp
if (sum < target)
{
    left++;
}
else if (sum > target)
{
    right--;
}
else
{
    return sum;
}
```

- `sum < target`：和太小，往右移动 `left`
- `sum > target`：和太大，往左移动 `right`
- `sum == target`：已经最优，直接返回

## 为什么不能只看差值来移动

例如：

- `target = 1`
- `sum = -4`
- 差值是 `5`

另一种情况：

- `target = 1`
- `sum = 6`
- 差值也是 `5`

虽然差值一样，但移动方向完全不同：

- `-4 < 1`，要让和变大，所以 `left++`
- `6 > 1`，要让和变小，所以 `right--`

这说明：

- 差值只表示距离
- `sum` 和 `target` 的大小关系才表示方向

## 更快的剪枝优化版

这是在标准双指针基础上继续优化常数的版本：

```csharp
public class Solution {
    public int ThreeSumClosest(int[] nums, int target) {
        Array.Sort(nums);
        int n=nums.Length,ans=int.MaxValue/2;
        for(int i=0;i<n-2;i++){
            if(i>0&&nums[i]==nums[i-1])continue;
            int sum=nums[i]+nums[i+1]+nums[i+2];
            if(sum>target){
                if(sum-target<Math.Abs(ans-target)){
                    ans=sum;
                }
                break;
            }
            sum=nums[i]+nums[n-1]+nums[n-2];
            if(sum<target){
                if(target-sum<Math.Abs(ans-target)){
                    ans=sum;
                }
                continue;
            }
            int j=i+1,k=n-1;
            while(j<k){
                sum=nums[i]+nums[j]+nums[k];
                if(sum==target){
                    return target;
                }
                if(Math.Abs(sum-target)<Math.Abs(ans-target)){
                    ans=sum;
                }
                if(sum>target){
                    k--;
                }else{
                    j++;
                }
            }
        }
        return ans;
    }
}
```

## 这个“更快版本”为什么快

它本质上还是：

- 排序
- 固定一个数
- 双指针

并没有换算法，复杂度仍然是：

```text
O(n^2)
```

它更快的原因是加了剪枝。

### 剪枝 1：当前最小和已经大于 `target`

```csharp
sum = nums[i] + nums[i + 1] + nums[i + 2];
if (sum > target)
{
    ...
    break;
}
```

这里的 `sum` 是当前固定 `nums[i]` 后能得到的最小三数和。

如果这个最小和都已经大于 `target`，说明：

- 后面再换更大的 `i`
- 和只会更大
- 不可能再更接近 target

所以可以直接：

- 更新答案
- 然后 `break`

### 剪枝 2：当前最大和仍然小于 `target`

```csharp
sum = nums[i] + nums[n - 1] + nums[n - 2];
if (sum < target)
{
    ...
    continue;
}
```

这里的 `sum` 是当前固定 `nums[i]` 后能得到的最大三数和。

如果这个最大和都还小于 `target`，说明：

- 这轮不管双指针怎么动
- 和都不可能达到 target

那么这一轮里最接近 target 的，只能是这个最大和。

所以可以：

- 更新答案
- 然后 `continue`

## 为什么这个版本仍然属于同一种最优解

- 算法级别没有变化，还是 `O(n^2)`
- 它只是：
  - 在最优思路上做了更强的常数优化
  - 让很多不必要的双指针过程直接跳过

所以可以理解成：

- 标准版：最优思路
- 剪枝版：最优思路 + 更快的工程化实现

## 复杂度

- 时间复杂度：`O(n^2)`
- 空间复杂度：`O(1)`，不算排序

## 复盘

- 这题和 `15.三数之和` 很像，但目标不一样：
  - `15` 是找所有和为 `0` 的三元组
  - `16` 是找最接近 `target` 的一个和
- 以后看到“最接近 target”：
  - 更新答案看差值
  - 指针移动看方向
- 如果标准双指针已经写对，再去学剪枝优化。

## 一句话记忆

- 谁更接近就更新谁，和太小就加左指针，和太大就减右指针；剪枝只是让这套最优思路跑得更快。
