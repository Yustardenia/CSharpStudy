# 14.最长公共前缀

## 题目类型

- 字符串
- 按列比较

## 我的当前状态

- 这题思路方向已经对了，已经想到“按列比较”。
- 现在的问题不是完全不会，而是代码实现里把“列索引”和“字符串索引”混了。

## 我这次代码里暴露的问题

你这次写的是：

```csharp
using System.Text;

public class Solution {
    public string LongestCommonPrefix(string[] strs)
    {
        StringBuilder ans = new StringBuilder();
        char ch = strs[0][0];
        int nowIndex = 0;
        int minLen = 0;
        foreach (var str in strs)
        {
            minLen = Math.Min(str.Length, minLen);
        }
        
        for (int i = 0; i < minLen; i++)
        {
            for (int j = 0; j < strs.Length; j++)
            {
                if (strs[j][i] != ch)
                {
                    return "";
                }
                
            }

            ans.Append(ch);
            ch = strs[nowIndex++][0];
        }
        return ans.ToString();
    }
}
```

### 错误 1：`minLen` 初始化错了

```csharp
int minLen = 0;
```

然后又写：

```csharp
minLen = Math.Min(str.Length, minLen);
```

这会导致：

- `minLen` 永远都是 `0`

因为任何正数和 `0` 取最小值，结果都还是 `0`。

所以后面的：

```csharp
for (int i = 0; i < minLen; i++)
```

根本不会进入循环。

### 错误 2：`ch` 只取了第一个字符

```csharp
char ch = strs[0][0];
```

你这题是要“按列比较”，所以每一轮外层循环都应该比较：

```csharp
strs[0][i]
```

而不是一开始只取第一个字符，再试图后面手动更新。

### 错误 3：发现不匹配时直接返回空串

```csharp
if (strs[j][i] != ch)
{
    return "";
}
```

这不对。

因为“不匹配”只说明：

- 当前这一列不再属于公共前缀

但前面已经匹配上的部分，仍然应该保留。

例如：

```text
["flower","flow","flight"]
```

在第 2 列才第一次不匹配，这时答案应该是 `"fl"`，不是空串。

### 错误 4：把列索引和字符串索引混了

```csharp
ch = strs[nowIndex++][0];
```

这一句的问题是：

- 你在换“字符串”
- 但这题真正应该换的是“列”

也就是说：

- 外层循环的 `i` 才是当前列号
- `ch` 应该是 `strs[0][i]`
- 不是不断去取不同字符串的第 `0` 个字符

## 正确思路

- 先找出最短字符串长度 `minLen`
- 因为公共前缀不可能比最短字符串更长
- 然后按列比较：
  - 先拿第一个字符串的第 `i` 个字符作为基准
  - 再看所有字符串第 `i` 个字符是否一致
- 一旦某一列不一致，就返回前面已经匹配成功的部分

## 标准写法

```csharp
public class Solution
{
    public string LongestCommonPrefix(string[] strs)
    {
        if (strs == null || strs.Length == 0)
        {
            return "";
        }

        int minLen = strs[0].Length;
        foreach (string str in strs)
        {
            minLen = Math.Min(minLen, str.Length);
        }

        for (int i = 0; i < minLen; i++)
        {
            char ch = strs[0][i];

            for (int j = 1; j < strs.Length; j++)
            {
                if (strs[j][i] != ch)
                {
                    return strs[0].Substring(0, i);
                }
            }
        }

        return strs[0].Substring(0, minLen);
    }
}
```

## 为什么这样写

### 1. 先找最短长度

```csharp
int minLen = strs[0].Length;
```

- 公共前缀最多只能和最短字符串一样长。

### 2. 外层循环按列走

```csharp
for (int i = 0; i < minLen; i++)
```

- `i` 表示当前在检查第几列。

### 3. 先拿第一个字符串当前列作基准

```csharp
char ch = strs[0][i];
```

- 其他字符串都拿这一列来和它比较。

### 4. 内层循环检查所有字符串

```csharp
for (int j = 1; j < strs.Length; j++)
```

- 让后面的每个字符串都检查当前列是否相同。

### 5. 发现不一致就立刻返回

```csharp
return strs[0].Substring(0, i);
```

- 因为前面 `0 ~ i-1` 这一段就是最长公共前缀。

## 复杂度

- 时间复杂度：`O(n * m)`
  - `n` 是字符串数量
  - `m` 是最短字符串长度
- 空间复杂度：`O(1)`

## 复盘

- 这题最重要的是：
  - 外层按列
  - 内层按字符串
- 以后看到“公共前缀”，先想：
  - 第 0 列是否都一样
  - 第 1 列是否都一样
  - 直到某一列第一次不一样

## 一句话记忆

- 固定列去比所有字符串，一列一列走，第一次不一样时前面那段就是答案。
