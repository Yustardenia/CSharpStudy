# 11.盛最多水的容器

## 题目类型

- 双指针
- 贪心

## 当前状态

- 这题我已经做对了。
- 当前写法就是标准最优解。
- 这题重点不是背代码，而是记住“为什么一定移动短板”。

## 我的解法

```csharp
public class Solution {
    public int MaxArea(int[] height)
    {
        int ans = 0;
        int i = 0;
        int j = height.Length - 1;
        while (i < j)
        {
            int h = Math.Min(height[i], height[j]);
            int s = j - i;
            int area = h * s;
            ans = Math.Max(ans, area);
            if (h == height[i])
            {
                i++;
            }

            else
            {
                j--;
            }

        }
        return ans;
    }
}
```

## 为什么这个解法对

当前这两个指针 `i` 和 `j` 能形成的面积是：

```text
面积 = 宽度 * 高度
     = (j - i) * min(height[i], height[j])
```

其中：

- 宽度由两根线的距离决定
- 高度由两边较短的那根线决定

所以真正限制面积的，是短板。

## 为什么只能移动短板

假设：

- 左边短
- 右边高

如果这时去移动右边：

- 宽度会变小
- 但高度上限仍然被左边短板限制

那么面积不可能变得更大。

只有移动短板，才有机会：

- 虽然宽度变小了
- 但遇到更高的线
- 从而让 `min(height[i], height[j])` 变大

这才可能产生更优答案。

## 代码里每一步在做什么

```csharp
int ans = 0;
```

- 记录当前最大面积。

```csharp
int i = 0;
int j = height.Length - 1;
```

- 左指针从最左边开始
- 右指针从最右边开始

```csharp
while (i < j)
```

- 只要两根线还没相遇，就继续尝试。

```csharp
int h = Math.Min(height[i], height[j]);
int s = j - i;
int area = h * s;
```

- `h` 是当前容器高度，也就是较短那根线
- `s` 是当前宽度
- `area` 是当前面积

```csharp
ans = Math.Max(ans, area);
```

- 更新全局最大值。

```csharp
if (h == height[i])
{
    i++;
}
else
{
    j--;
}
```

- 如果左边是短板，就移动左边
- 否则移动右边

这里其实就是在做：

- 移动较短的那一边

## 一个小补充

更常见的写法是：

```csharp
if (height[i] < height[j])
{
    i++;
}
else
{
    j--;
}
```

你的写法：

```csharp
if (h == height[i])
```

本质上也是同一个意思。

区别只是：

- 当两边一样高时
  - 你的写法移动左边
  - 另一种常见写法移动右边

这两种都对，因为相等时移动哪边都可以。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 复盘

- 以后看到这种“两端向中间收缩”的题，先想双指针。
- 看到面积公式里有 `min(a, b)`，就要警觉：
  - 真正限制答案的可能是较小值
  - 这类题往往应该移动短板

## 一句话记忆

- 宽度一定会缩小，所以只能寄希望于短板变高，因此每次移动短板。
