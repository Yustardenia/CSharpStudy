# 15.三数之和

## 题目类型

- 排序
- 双指针
- 去重

## 我的当前状态

- 这题一开始会先想到三重循环。
- 现在已经写到了标准的“排序 + 固定一个数 + 双指针”版本。
- 当前真正暴露出来的问题，主要是边界条件和去重细节。

## 我一开始的朴素想法

- 枚举三个数，看它们的和是不是 `0`
- 这个想法能帮助理解题意，但问题很多：
  - 下标可能重复
  - 三元组顺序不同会造成重复答案
  - 三重循环复杂度太高，是 `O(n^3)`

所以这题真正要掌握的是：

- 先排序
- 固定第一个数
- 后面两个数用双指针去找

## 我的双指针版本

```csharp
public class Solution
{
    public IList<IList<int>> ThreeSum(int[] nums)
    {
        Array.Sort(nums);
        IList<IList<int>> ans = new List<IList<int>>();
        for (int i = 0; i < nums.Length - 2; i++)
        {
            if (nums[i] >= 0)
            {
                break;
            }

            if (i > 0 && nums[i] == nums[i-1])
            {
                continue;
            }

            int left = i + 1;
            int right = nums.Length - 1;
            while (left < right)
            {
                if (nums[i] + nums[left] + nums[right] == 0)
                {
                    ans.Add(new List<int>()
                    {
                        nums[i],nums[left],nums[right]
                    });
                    left++;
                    right--;
                    while (left < right && nums[left] == nums[left-1])
                    {
                        left++;
                    }
                    while(left < right && nums[right] == nums[right+1])
                    {
                        right--;
                    }
                }

                else if (nums[i] + nums[left] + nums[right] < 0)
                {
                    left++;
                }

                else 
                {
                    right--;
                }
            }
        }
        
        return ans;
    }
}
```

## 这版错在哪

### 错误点：这里不能写 `>= 0`

```csharp
if (nums[i] >= 0)
{
    break;
}
```

应该改成：

```csharp
if (nums[i] > 0)
{
    break;
}
```

## 为什么一定要写 `> 0`

因为当 `nums[i] == 0` 时，仍然有可能存在答案：

```text
[0, 0, 0]
```

例如：

```text
nums = [0,0,0]
```

排序后还是：

```text
[0,0,0]
```

如果你写：

```csharp
if (nums[i] >= 0)
{
    break;
}
```

那么在 `i = 0` 时就直接退出了，结果会漏掉：

```text
[[0,0,0]]
```

而正确逻辑应该是：

- 只有当 `nums[i] > 0` 时
- 当前数已经是正数
- 后面只会更大
- 三个数加起来不可能再等于 `0`

这时候才可以提前结束。

## 正确思路

### 1. 先排序

排序后相同元素会靠在一起，方便：

- 双指针移动
- 去重

### 2. 固定第一个数 `nums[i]`

把问题转成：

- 在右边区间找两个数
- 使得：

```text
nums[left] + nums[right] = -nums[i]
```

### 3. 双指针找另外两个数

- `left = i + 1`
- `right = nums.Length - 1`

如果当前和：

- 小于 `0`，说明太小，`left++`
- 大于 `0`，说明太大，`right--`
- 等于 `0`，加入答案，然后跳过重复值

## 标准写法

```csharp
public class Solution
{
    public IList<IList<int>> ThreeSum(int[] nums)
    {
        Array.Sort(nums);
        IList<IList<int>> ans = new List<IList<int>>();

        for (int i = 0; i < nums.Length - 2; i++)
        {
            if (nums[i] > 0)
            {
                break;
            }

            if (i > 0 && nums[i] == nums[i - 1])
            {
                continue;
            }

            int left = i + 1;
            int right = nums.Length - 1;

            while (left < right)
            {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum == 0)
                {
                    ans.Add(new List<int> { nums[i], nums[left], nums[right] });

                    left++;
                    right--;

                    while (left < right && nums[left] == nums[left - 1])
                    {
                        left++;
                    }

                    while (left < right && nums[right] == nums[right + 1])
                    {
                        right--;
                    }
                }
                else if (sum < 0)
                {
                    left++;
                }
                else
                {
                    right--;
                }
            }
        }

        return ans;
    }
}
```

## 为什么这样去重

### 去重 `i`

```csharp
if (i > 0 && nums[i] == nums[i - 1])
{
    continue;
}
```

- 如果当前固定数和前一个一样，就跳过
- 否则会得到重复三元组

### 找到答案后去重 `left` 和 `right`

```csharp
while (left < right && nums[left] == nums[left - 1])
{
    left++;
}

while (left < right && nums[right] == nums[right + 1])
{
    right--;
}
```

- 因为当前这组答案已经记录过了
- 相同的值继续用，只会得到重复答案

## 复杂度

- 时间复杂度：`O(n^2)`
- 空间复杂度：`O(1)`，不算返回结果

## 复盘

- 这题核心不是“三个循环凑和”，而是“排序后把两数之和嵌进来”。
- 看到“三元组 + 去重”，优先想：
  - 排序
  - 固定一个数
  - 剩下两个数双指针
- 以后看到提前结束条件，要特别小心：
  - `> 0` 和 `>= 0` 这种地方很容易少一组答案

## 一句话记忆

- 排序后固定一个数，剩下两个数左右夹，找到答案后别忘了三处去重。
